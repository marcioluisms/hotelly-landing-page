# Perguntas frequentes sobre configurações gerais

## O Hotelly funciona para hotéis, pousadas, hostels e chalés?

Sim. O Hotelly foi projetado para qualquer tipo de meio de hospedagem independente: pousadas, hotéis boutique, chalés, hostels e até bed & breakfasts. O sistema se adapta ao porte — de 1 quarto a 50 quartos.

## Posso conectar com Booking, Airbnb ou outras OTAs?

No momento, o Hub de distribuição (channel manager) está no roadmap futuro. Será desenvolvido quando a demanda justificar. Hoje, o Hotelly trabalha com reservas diretas (WhatsApp + Motor de Reservas no site).

## O que é o Mercado Pago? Posso usar outro gateway?

O Mercado Pago é o gateway de pagamento integrado ao Hotelly para receber pagamentos dos hóspedes (Pix, cartão de crédito, etc.). Atualmente é o único gateway suportado para pagamentos automatizados. Pagamentos manuais (dinheiro, cartão na máquina física) podem ser registrados sem gateway.

## Como funciona o onboarding?

O onboarding é assistido pela equipe Hotelly. Não é self-service. A equipe acompanha você no cadastro de quartos, tipos, preços e políticas. Durante a fase de Parceiro Fundador, você tem canal direto com o fundador.

## Meus dados estão seguros?

Sim. O Hotelly usa criptografia AES-256-GCM para credenciais sensíveis, tokens JWT para autenticação, e dados pessoais (PII) nunca aparecem em logs do sistema. A Trilha de Auditoria registra todas as ações sensíveis para rastreabilidade.

## Posso usar o Hotelly no celular?

Sim. O painel funciona no navegador de qualquer dispositivo (computador, tablet ou celular). Não há aplicativo dedicado — o acesso é pela web, o que garante que você sempre tem a versão mais atualizada.

## Onde ficam meus dados se eu cancelar?

Os dados são preservados mesmo após o cancelamento da assinatura. A política de retenção garante que seu histórico de reservas, hóspedes e financeiro não é deletado.

## O que acontece se a internet da pousada cair?

O Concierge IA continua funcionando (roda nos servidores do Hotelly). Apenas o acesso ao dashboard fica indisponível até a internet voltar. Dados não são perdidos — quando a conexão restabelecer, tudo estará atualizado.
